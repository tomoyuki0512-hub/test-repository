# ローカルサーバーの起動方法

FigmaにHTMLデザインを取り込む際などに使うローカルサーバーの起動方法まとめ。

---

## 必要な環境のインストール

### Node.js

#### バージョン確認

```bash
node -v
npm -v
```

インストール済みであればバージョン番号が表示される（例：`v20.11.0`）。
コマンドが見つからないと表示された場合はインストールが必要。

#### インストール方法

- 公式サイト（https://nodejs.org）から **LTS版** をダウンロードしてインストール
- LTS版は安定版なので通常こちらを選ぶ
- インストール後、ターミナル（またはコマンドプロンプト）を再起動してからバージョン確認を行う

---

### Python3

#### バージョン確認

```bash
python3 --version
```

Windowsの場合は以下も試す：

```bash
python --version
```

インストール済みであればバージョン番号が表示される（例：`Python 3.11.4`）。

#### インストール方法

- 公式サイト（https://www.python.org）から最新版をダウンロードしてインストール
- Windowsの場合、インストール時に **「Add Python to PATH」にチェックを入れる**（重要）
- インストール後、ターミナルを再起動してからバージョン確認を行う

---

## 2つのコマンドの特徴比較

| | `npx serve .` | `python3 -m http.server 8080` |
|---|---|---|
| 必要環境 | Node.js | Python3 |
| デフォルトポート | 3000 | 指定が必要（例：8080） |
| 速度 | やや速い | 標準的 |
| 設定のしやすさ | オプションが豊富 | シンプル |
| 向いている用途 | フロントエンド開発向け | 手軽な確認向け |

どちらも機能的には大差なく、インストールされている方を使えばよい。

---

## どこのフォルダで実行する？

**HTMLファイルがあるフォルダ**で実行する。

```
例：C:\Users\yourname\Desktop\mysite\
    ├── index.html
    ├── style.css
    └── script.js
```

```bash
cd C:\Users\yourname\Desktop\mysite
```

---

## 起動コマンド

### Node.js を使う場合

```bash
npx serve .
```

- デフォルトで `http://localhost:3000` で起動

### Python3 を使う場合

```bash
python3 -m http.server 8080
```

- `http://localhost:8080` で起動

どちらもHTMLファイルがあるフォルダで実行するだけでOK。

---

## フォルダ構成

特に決まりはないが、`index.html` があると自動的にトップページとして認識される。

```
mysite/
├── index.html   ← http://localhost:3000 でアクセスできる
├── other.html   ← http://localhost:3000/other.html でアクセス
├── style.css
└── script.js
```

---

## そもそも何をしているのか？

HTMLファイルをダブルクリックで開くと `file://` プロトコルで開く。これだと：

- 外部からアクセスできない
- 一部のJavaScriptや拡張機能が動かない（例：Builder.ioのChrome拡張）

ローカルサーバーを起動すると `http://localhost:XXXX` というURLでアクセスできるようになり、本物のWebサーバーと同じ環境になる。

---

## 用途：あくまでローカル確認用

**これらはローカル専用のサーバーであり、一般公開には使えない。**

理由：
- PCの電源を切ったら止まる
- セキュリティ対策がない（外部公開は危険）
- 固定IPがないためURLが変わる

### 用途の整理

```
ローカルサーバー → 自分だけが見るサンプル確認・開発中のプレビュー用
一般公開        → 別のホスティングサービスが必要
```

### 小規模HPを一般公開したい場合

| サービス | 費用 | 特徴 |
|---|---|---|
| GitHub Pages | 無料 | 静的サイト向け、GitHubと連携 |
| Netlify | 無料プランあり | デプロイが簡単、フォーム機能あり |
| Vercel | 無料プランあり | Next.js等との相性が良い |
| さくらインターネット | 有料 | 日本語サポートあり、安定 |

## スマホからアクセスする場合

PCとスマホが同じWi-Fiに接続されていれば、スマホのブラウザからもアクセスできる。

1. PCのIPアドレスを確認する
   - Windows: `ipconfig` コマンド → IPv4アドレスを確認
   - Mac/Linux: `ifconfig` または `ip a` コマンド
2. スマホのブラウザで以下にアクセス：
   ```
   http://192.168.1.10:3000   ← PCのIPアドレスに置き換える
   ```

---

## IPアドレスを毎回入力するのが面倒な場合

### 方法1：ホスト名（.local）を使う（設定不要）

IPアドレスの代わりにPCの名前でアクセスできる。

```
http://PCの名前.local:3000
```

現在のホスト名の確認方法：

```bash
hostname
```

- **Mac** → そのまま使える（Bonjourが標準搭載）
- **Windows** → Windows10/11なら多くの場合そのまま使える

---

### 方法2：ホスト名を変更する

長い・わかりにくいホスト名を短く変えると入力が楽になる。

#### Windowsの場合

1. 「スタート」→「設定」→「システム」→「詳細情報」を開く
2. 「このPCの名前を変更する」をクリック
3. 新しい名前を入力（例：`mypc`）
4. 再起動して反映

変更後のアクセス例：
```
http://mypc.local:3000
```

#### Macの場合

1. 「システム設定」→「一般」→「情報」を開く
2. 「ローカルホスト名」の横の「編集」をクリック
3. 新しい名前を入力（例：`mypc`）
4. 「完了」をクリック（再起動不要）

または、ターミナルから変更する場合：

```bash
sudo scutil --set HostName mypc
sudo scutil --set LocalHostName mypc
sudo scutil --set ComputerName mypc
```

変更後のアクセス例：
```
http://mypc.local:3000
```

---

### 方法3：ngrokを使う（ネット経由で共有したい場合）

ローカルサーバーに一時的な固定URLを発行できるツール。

```bash
npx ngrok http 3000
```

実行すると以下のようなURLが発行される：
```
https://abc123.ngrok.io  ← このURLをスマホで開くだけ
```

- 設定不要・URLは毎回変わるが入力は楽
- 無料プランあり
- 同じWi-Fi外（外出先など）からもアクセスできる

---

### 方法4：ルーターでIPを固定する（恒久的な解決策）

ルーターの設定画面でPCのMACアドレスに対して常に同じIPを割り当てる（DHCP予約）。
一度設定すれば **IPが変わらなくなる** のでブックマーク登録が使える。

設定方法はルーターのメーカーによって異なるため、取扱説明書を参照。

---

### 方法の比較

| 方法 | 手間 | 効果 |
|---|---|---|
| ホスト名（.local） | なし | 環境によっては使えない |
| ホスト名を変更 | 一度だけ | 短い名前で固定できる |
| ngrok | 少 | 確実・毎回URL変わる |
| ルーターでIP固定 | 一度だけ | 恒久的にIPが変わらない |
