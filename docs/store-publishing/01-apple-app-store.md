# Apple App Store 公開フロー ゼロからガイド

iOS / iPad アプリを **App Store に公開するまで** を、初めての人でも迷わないように順を追って解説します。
（→ Android 版は [Google Play 公開フロー](02-google-play.md)、収益化は [収益化パターン](03-monetization-ads-subscription.md)）

---

## 0. 始める前に：必要なもの

| 必要なもの | 内容 | 費用 |
|---|---|---|
| **Mac** | Xcode（Apple公式の開発ツール）が Mac でしか動かないため、実質必須 | 手持ちのMac / クラウドMac |
| **Apple ID** | 普段のApple ID。これを開発者登録に使う | 無料 |
| **Apple Developer Program 登録** | アプリ公開に必須の有料会員 | **年 99 USD** |
| **iPhone/iPad 実機**（推奨） | 実機テスト用。シミュレータだけでは不十分な機能もある | 手持ち端末 |

> **ポイント:** 開発者登録は **年額制（毎年更新）**。更新を忘れると公開中のアプリがストアから消えるので注意。

---

## 1. 全体の流れ（7ステップ）

```
① Apple Developer Program 登録
        ▼
② 証明書・識別子・プロファイルの準備（署名まわり）
        ▼
③ App Store Connect でアプリ枠を作成
        ▼
④ アプリをビルドしてアップロード（Xcode / Transporter）
        ▼
⑤ ストア掲載情報を入力（説明・スクショ・価格・年齢・プライバシー）
        ▼
⑥ 審査に提出（App Review）
        ▼
⑦ 公開（自動 or 手動リリース）→ 運用・アップデート
```

---

## 2. ステップ①：Apple Developer Program に登録

1. [developer.apple.com](https://developer.apple.com/) にApple IDでサインイン
2. 「Account」→「Enroll（登録）」へ
3. **個人（Individual）** か **法人（Organization）** を選ぶ
   - **個人**: 本人確認のみ。ストア上の販売者名＝個人名になる
   - **法人**: 会社として公開できるが、**D-U-N-S番号**（企業識別番号）が必要。取得に数日〜2週間かかることも
4. 支払い（年99USD）を済ませる
5. 審査・本人確認が通ると登録完了（通常数十分〜2日程度）

> **法人で出すなら D-U-N-S 番号の取得を最優先で。** これが全体のボトルネックになりがち。

---

## 3. ステップ②：署名まわりの準備（証明書・App ID・プロビジョニング）

「このアプリは確かにあなたが作った」と証明する仕組みです。難しく見えますが、**Xcodeの自動署名（Automatically manage signing）を使えば大半は自動**でやってくれます。

| 用語 | 何のため | 補足 |
|---|---|---|
| 証明書（Certificate） | 開発者本人の電子的な身分証 | 配布用（Distribution）が公開に必要 |
| App ID（Bundle ID） | アプリ固有の識別子（例 `com.example.myapp`） | **一度決めたら変更不可**。慎重に |
| プロビジョニングプロファイル | 「どの証明書で・どのアプリを・どの端末で」動かせるかの許可証 | 自動署名なら自動生成 |

**初心者の進め方:** Xcode で対象アプリを開き、`Signing & Capabilities` タブで自分のチーム（Developer登録したアカウント）を選び、「Automatically manage signing」にチェック。これでOK。

---

## 4. ステップ③：App Store Connect でアプリ枠を作成

[App Store Connect](https://appstoreconnect.apple.com/) はAppleのアプリ管理画面です。

1. 「マイApp」→「＋」→「新規App」
2. 入力項目:
   - **プラットフォーム**（iOS など）
   - **名前**（ストアに表示されるアプリ名。30文字以内）
   - **プライマリ言語**
   - **バンドルID**（ステップ②で作ったものを選択）
   - **SKU**（自分用の管理番号。任意の文字列でOK）
3. これで「空のアプリ枠」ができ、ここに情報とビルドを入れていく

---

## 5. ステップ④：ビルドしてアップロード

1. Xcode でアプリのバージョン番号・ビルド番号を設定
2. 端末ターゲットを「Any iOS Device」にする
3. メニュー `Product → Archive` でアーカイブ（配布用ビルド）を作成
4. Organizer 画面から `Distribute App → App Store Connect → Upload`
5. アップロード後、App Store Connect 側で「ビルドが処理中」→ 数分〜数十分で選択可能になる

> Flutter の場合は `flutter build ipa` でアーカイブを作り、Xcode または **Transporter** アプリでアップロードする方法もあります。

### TestFlight（任意だが推奨）

アップロードしたビルドは **TestFlight** で社内・外部テスターに配布できます。本番公開前に実機で動作確認できるので、活用を強く推奨します。

---

## 6. ステップ⑤：ストア掲載情報を入力

審査前に、ユーザーに見える情報をそろえます。

| 項目 | 内容・注意 |
|---|---|
| **アプリ名 / サブタイトル** | 検索にも影響。誇大・他社名の無断使用はNG |
| **説明文** | 機能を正確に。事実と異なる記載はリジェクト対象 |
| **キーワード** | 検索対策（100文字）。カンマ区切り |
| **スクリーンショット** | 必須。**指定サイズ**（6.9インチ/6.5インチiPhoneなど）を用意。実画面と乖離するとリジェクト |
| **プレビュー動画** | 任意 |
| **アイコン** | 1024×1024px。透過・角丸は不可（Appleが自動処理） |
| **カテゴリ** | プライマリ/セカンダリ |
| **年齢制限（レーティング）** | 質問に回答して自動判定 |
| **価格・販売地域** | 無料/有料、配信する国を選択 |
| **プライバシーポリシーURL** | **必須**。Webに公開したURLが要る |
| **App プライバシー（収集データ申告）** | 「どんなデータを何の目的で集めるか」を申告。広告SDK利用時は特に正確に |
| **サポートURL / 連絡先** | 必須 |

### アプリ内課金がある場合（重要）

サブスクや課金アイテムを売るなら、**App Store Connect で商品（プロダクト）を別途登録**し、最初の審査では「アプリ本体と一緒に商品も提出」します。詳細は [収益化パターン](03-monetization-ads-subscription.md)。

---

## 7. ステップ⑥：審査に提出（App Review）

1. 使用するビルドを選択
2. 「審査用情報」を記入
   - **テスト用アカウント**（ログインが必要なアプリは必須。これが無いとほぼリジェクト）
   - 審査担当者への補足メモ（特殊な使い方があれば説明）
3. 「審査へ提出」

### 審査でよくあるリジェクト理由（先に対策）

| 理由 | 対策 |
|---|---|
| ログイン情報がなく動作確認できない | テスト用アカウントを必ず提供 |
| クラッシュ・バグ | 実機で十分テスト |
| 機能が薄い・Webサイトをそのまま包んだだけ | アプリならではの価値を持たせる |
| 課金を外部決済に誘導している | デジタル商品はApple IAPを使う（規約 3.1.1） |
| プライバシー申告と実際の挙動が不一致 | 申告内容を実装と一致させる |
| 「購入の復元」ボタンが無い | サブスク・買い切りには復元導線を実装 |
| 第三者ログインがあるのに Sign in with Apple が無い | 要件を満たすよう実装 |

> 審査期間は **おおむね24〜48時間**（混雑期や初回はもっとかかることも）。リジェクトされても理由が来るので、直して再提出すればOK。

---

## 8. ステップ⑦：公開とリリース方法

審査通過後の公開方法を選べます。

| リリース方法 | 内容 |
|---|---|
| 自動リリース | 審査通過後ただちに公開 |
| 手動リリース | 通過後、自分のタイミングで公開ボタンを押す（マーケ施策と合わせやすい） |
| 段階的リリース（Phased Release） | アップデート時に**7日かけて0%→100%**へ徐々に配信。不具合の影響を抑える |

---

## 9. 公開後の運用・アップデート

- **アップデート**: 新バージョンを作って同じ流れ（ビルド→情報更新→審査→公開）。**毎回審査がある**
- **指標確認**: App Store Connect / App Analytics でDL数・売上・継続率を確認
- **規約・要件追従**: AppleはたびたびSDK・APIの最低要件（例: 最新Xcode/SDKでのビルド義務）を更新。期限までに対応しないと新規提出ができなくなる
- **返金対応**: 返金はApple側で処理（開発者は原則関与しない）

---

## 10. 期間・費用まとめ（早見表）

| 項目 | 目安 |
|---|---|
| 開発者登録費 | 年 99 USD |
| 登録にかかる時間 | 個人:数十分〜2日 / 法人:D-U-N-S取得含め数日〜2週間 |
| 審査期間 | 約24〜48時間（変動あり） |
| 手数料 | 売上の **30%**（条件を満たすと **15%**：[後述](03-monetization-ads-subscription.md)） |
| 必要環境 | Mac + Xcode（実質必須） |
