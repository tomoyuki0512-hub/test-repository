# Apple vs Google 比較・公開前チェックリスト

両ストアに出す人向けの **早見表** と、提出前に確認すべき **チェックリスト** をまとめます。
（詳細は [Apple 公開フロー](01-apple-app-store.md) / [Google Play 公開フロー](02-google-play.md) / [収益化パターン](03-monetization-ads-subscription.md)）

---

## 1. Apple vs Google 早見表

| 項目 | Apple App Store | Google Play |
|---|---|---|
| 登録費 | 年 99 USD（毎年更新） | 25 USD（初回1回のみ） |
| 必要PC | Mac（実質必須） | Windows/Mac/Linux 可 |
| 管理画面 | App Store Connect | Google Play Console |
| 開発ツール | Xcode | Android Studio など |
| 配布形式 | IPA（Archive） | AAB（必須） |
| 署名 | 証明書＋プロビジョニング（Xcode自動署名可） | Play アプリ署名（Googleが鍵を保管） |
| 審査 | 人による審査中心（約24〜48時間） | 自動＋人（数時間〜数日、初回長め） |
| テスト配布 | TestFlight | 内部/クローズド/オープンテスト |
| 段階的公開 | Phased Release（7日） | Staged rollout（％指定、停止可能） |
| 手数料 | 30%（小規模/継続で15%） | 30%（年100万USDまで/定期購入15%） |
| プライバシー申告 | App プライバシー | データセーフティ |
| 法人登録 | D-U-N-S番号が必要 | D-U-N-S番号が必要 |
| 厳しさの傾向 | リジェクトされやすい | 規約違反に厳格（特に権限・申告整合） |

## 2. 共通の必須準備（両ストアで要る）

- [ ] アプリアイコン（高解像度）
- [ ] スクリーンショット（各指定サイズ）
- [ ] アプリ名・説明文・キーワード
- [ ] **プライバシーポリシーのURL**（Webに公開済み）
- [ ] サポート連絡先（メール等）
- [ ] 年齢レーティングの回答
- [ ] **収集データの申告**（広告SDK利用なら特に正確に）
- [ ] ログインが必要なら **テスト用アカウント**
- [ ] 課金がある場合は **商品（サブスク/IAP）の登録**
- [ ] 課金がある場合は **「購入を復元」導線** の実装

---

## 3. Apple 提出前チェックリスト

- [ ] Apple Developer Program 登録済み（更新期限も確認）
- [ ] Bundle ID を確定（後から変更不可）
- [ ] 配布用証明書・プロファイルの準備（自動署名でも可）
- [ ] Archive をアップロード済み、ビルドが選択可能
- [ ] スクショが**現行の必須サイズ**を満たす
- [ ] App プライバシーを実装と一致させた
- [ ] 第三者ログインがあるなら Sign in with Apple を検討
- [ ] 審査メモ・テストアカウントを記入
- [ ] 自動/手動リリースを選択

## 4. Google Play 提出前チェックリスト

- [ ] デベロッパー登録・**本人確認**完了
- [ ] （新規個人なら）**クローズドテスト要件**を満たしたか確認
- [ ] アップロード鍵（keystore）をバックアップ
- [ ] **AAB** をアップロード済み
- [ ] データセーフティを実装と一致させた
- [ ] 権限は必要最小限か（デリケート権限の正当化）
- [ ] 対象APIレベル（targetSdkVersion）が**最新要件**を満たす
- [ ] フィーチャーグラフィックを用意
- [ ] 段階的公開の割合を設定

---

## 5. 公開後の継続タスク（両ストア共通）

- [ ] DL数・売上・継続率・クラッシュ率を定期確認
- [ ] OS/SDK 最低要件の更新に追従（期限あり）
- [ ] レビュー（口コミ）への返信・評価対策
- [ ] ポリシー改定の通知をチェック
- [ ] 収益化施策（価格・トライアル・広告頻度）のABテスト

---

## 6. つまずきやすいポイント Top5

1. **Mac が無くてiOSが出せない** → クラウドMac等を早めに用意
2. **法人のD-U-N-S番号取得待ち** → 最優先で着手
3. **Google新規個人のテスト要件**（人数×期間）→ スケジュールに織り込む
4. **プライバシー申告と実装の不一致** → 両ストア共通のリジェクト最頻原因
5. **手数料を30%と思い込み収益計画を誤る** → 15%優遇の対象か確認（[収益化](03-monetization-ads-subscription.md)）
