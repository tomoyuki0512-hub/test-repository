# アプリストア公開ガイド 総合インデックス（はじめに）

このドキュメント群は、**作ったアプリを Apple App Store（iOS/iPad）と Google Play（Android）で公開し、広告やサブスクで収益を得るまで** の流れを、**専門知識ゼロの人でも理解できる** ようにまとめたものです。

- 対象: これから初めてアプリを公開する個人・企業の担当者、企画・PM、開発者
- カバー範囲: **アカウント登録 → ビルド → ストア掲載情報 → 審査 → 公開 → 更新 → 収益化（広告・サブスク）**
- 立場を問わず読めるよう、専門用語は都度かみくだいて説明します

> このページだけ読めば「全体像」と「どこに何が書いてあるか」がわかります。詳細は各ページへ。

---

## 1. まず結論（30秒サマリ）

アプリ公開は、ざっくり次の5ステップです。**AppleもGoogleも流れは同じ**で、細かいルールと値段が違うだけです。

```
① 開発者登録   → ② アプリの準備   → ③ ストア情報の入力 → ④ 審査     → ⑤ 公開・運用
（お金を払って   （ビルド・署名・    （説明文・画像・     （ストアの    （アップデート・
  アカウント作成）  価格・年齢区分）    プライバシー申告）    人＆自動審査）  収益化・分析）
```

| | Apple App Store | Google Play |
|---|---|---|
| 登録費用 | **年 99 USD**（毎年更新） | **25 USD（初回1回のみ）** |
| 審査 | 人による審査が中心（**1日〜数日**） | 自動＋人による審査（**数時間〜数日、初回は長め**） |
| 公開までの体感 | 厳しめ・リジェクトされやすい | 比較的ゆるめだが規約違反に厳格 |
| 必須PC | **Mac が実質必須**（Xcode） | Windows/Mac/Linux いずれでも可 |

詳しくは → [Apple 公開フロー](01-apple-app-store.md) / [Google Play 公開フロー](02-google-play.md)

## 5. 最低限おさえる用語集（共通言語）

| 用語 | 読み / 英語 | 意味 |
|---|---|---|
| 開発者アカウント | Developer Account | アプリを公開するために必要な、ストア公式の登録アカウント |
| App Store Connect | — | Appleの管理画面。アプリ情報の入力・審査提出・売上確認を行う |
| Google Play Console | — | Googleの管理画面。Appleでいう App Store Connect に相当 |
| ビルド | Build | ソースコードを実際に動くアプリファイルに変換すること |
| 署名 | Signing | 「このアプリは確かに自分が作った」と証明する電子署名。ストア提出に必須 |
| 審査（レビュー） | Review | ストアがアプリの安全性・規約適合を確認する工程。合格しないと公開不可 |
| リジェクト | Reject | 審査で却下されること。理由が通知されるので、直して再提出する |
| IAP / アプリ内課金 | In-App Purchase | アプリ内でデジタル商品（サブスク・機能解放など）を売る仕組み |
| サブスクリプション | Subscription | 月額・年額など、定期的に自動課金される継続課金 |
| 手数料（コミッション） | Commission | 売上からストアが徴収する割合（おおむね 15〜30%） |
| 段階的公開 | Staged / Phased Release | いきなり全員でなく、一部ユーザーから徐々に公開する仕組み |
| プライバシー申告 | Privacy / Data Safety | 「どんな個人データを集めるか」をストアに申告すること。現在は両ストアで必須 |

## 6. よくある誤解（先に潰しておく）

- **「無料アプリなら登録費もタダ」→ ✕**：無料アプリでも開発者登録の費用（Apple年99$ / Google25$）は必要です。
- **「Macが無くてもiOSアプリは出せる」→ ほぼ✕**：iOSのビルド・署名・提出には実質Macが必要です（クラウドMacサービスで代替は可能）。
- **「審査は1回で必ず通る」→ ✕**：特にAppleはリジェクトが日常。理由を読んで直す前提でスケジュールを組みます。
- **「広告とサブスクはどちらか一方」→ ✕**：両方を組み合わせるハイブリッドが今の主流です（[収益化パターン](03-monetization-ads-subscription.md)）。
- **「公開したら終わり」→ ✕**：OS更新・規約変更・SDK要件で、定期的なアップデートが事実上必須です。
