# カスタムドメイン設定ガイド

GitHub Pagesに独自ドメイン（例：`www.mysite.com`）を設定する方法まとめ。

---

## カスタムドメインとは？

デフォルトのGitHub PagesのURLは：
```
https://ユーザー名.github.io/リポジトリ名/
```

カスタムドメインを設定すると：
```
https://www.mysite.com/
```
のような独自のURLで公開できる。

---

## ドメインの種類

| 種類 | 例 | 説明 |
|---|---|---|
| **apexドメイン** | `mysite.com` | `www`なしのルートドメイン |
| **サブドメイン** | `www.mysite.com` | `www`などのプレフィックス付き |
| **カスタムサブドメイン** | `blog.mysite.com` | 任意のプレフィックス |

wwwありとwwwなし、両方設定しておくのが一般的。

---

## ドメインの取得方法

ドメインはドメイン登録サービス（レジストラ）で購入する。

| サービス | 特徴 |
|---|---|
| **お名前.com** | 国内最大手・日本語サポートあり |
| **ムームードメイン** | GMO系・管理画面がシンプル |
| **Cloudflare Registrar** | 原価提供・管理機能が豊富 |
| **Google Domains** | シンプル・Googleサービスと連携 |
| **Namecheap** | 海外サービス・価格が安い |

`.com` ドメインで年間約1,000〜2,000円程度が相場。

---

## 設定手順

### ステップ1：CNAMEファイルをリポジトリに追加

`docs/` フォルダに `CNAME` というファイルを作成し、ドメイン名だけを記載する。

```
www.mysite.com
```

拡張子なし・1行だけ・改行なしで保存してプッシュする。

### ステップ2：GitHubのPages設定でカスタムドメインを入力

1. リポジトリ → Settings → Pages
2. 「Custom domain」にドメイン名を入力
3. 「Save」をクリック

### ステップ3：DNS設定（ドメイン登録サービス側の作業）

ドメイン登録サービスの管理画面でDNSレコードを設定する。

#### wwwありサブドメインの場合（推奨）

CNAMEレコードを追加：

| 種別 | ホスト名 | 値 |
|---|---|---|
| CNAME | www | ユーザー名.github.io |

#### apexドメイン（wwwなし）の場合

AレコードをGitHubのIPアドレスに向ける：

| 種別 | ホスト名 | 値 |
|---|---|---|
| A | @ | 185.199.108.153 |
| A | @ | 185.199.109.153 |
| A | @ | 185.199.110.153 |
| A | @ | 185.199.111.153 |

#### 両方設定する場合（wwwあり・なし両対応）

上記のCNAMEレコードとAレコードを両方追加する。

---

## HTTPSを有効にする

カスタムドメイン設定後、GitHub Pagesで無料のHTTPS（SSL）を有効化できる。

1. Settings → Pages
2. DNS設定が反映されるまで待つ（数分〜数時間）
3. 「Enforce HTTPS」にチェックを入れる

---

## DNS反映の待ち時間

DNS設定の変更は即座には反映されない。

| 目安 | 状況 |
|---|---|
| 数分〜1時間 | 多くの場合はこの範囲で反映 |
| 最大48時間 | 世界中への完全な反映には最大この時間 |

反映前にアクセスすると「サイトが見つからない」と表示されるが、待てば解決する。

---

## 注意点

- CNAME ファイルは `docs/` フォルダ内に置く（GitHub Pagesのソースフォルダと同じ場所）
- カスタムドメインを削除するとデフォルトの `github.io` URLに戻る
- 無料プランでもカスタムドメインは使用可能（PublicリポジトリならOK）

---

## トラブルシューティング

| 症状 | 原因 | 対処 |
|---|---|---|
| サイトが表示されない | DNS未反映 | 最大48時間待つ |
| HTTPSが有効にならない | DNS設定ミス | AレコードまたはCNAMEを再確認 |
| 別のURLにリダイレクトされる | CNAMEファイルの内容ミス | ドメイン名だけ記載されているか確認 |
| 「ドメインが使用中」エラー | 他のリポジトリで設定済み | 他のリポジトリのPages設定を確認 |
