# ケース面接 完全ガイド（内容と対策）

コンサルティングファーム（マッキンゼー・BCG・ベイン等）や外資系企業、総合商社・事業会社の一部で実施される「ケース面接」について、**出題される内容**と**具体的な対策方法**をまとめます。

> 覚えておくと役立つ各種の数値・データは別ページにまとめています。
> → [ケース面接で覚えておくべき数値まとめ](case-interview-key-numbers.md)

---

## 1. ケース面接とは

ケース面接とは、**ビジネス上の課題（お題）を面接官から与えられ、その場で構造的に分析し、解決策や結論を導くプロセス**を評価する面接です。答えそのものよりも、**思考プロセス・論理性・コミュニケーション**を見られます。

### 評価されるポイント

| 観点 | 見られていること |
| --- | --- |
| 構造化力 | 問題をMECEに分解し、論点を整理できるか |
| 論理的思考 | 因果関係・数字の裏付けを持って主張できるか |
| 定量感覚 | 概算（フェルミ推定）を素早く正確に行えるか |
| ビジネス感覚 | 現実的で実行可能な打ち手を出せるか |
| コミュニケーション | 結論から簡潔に、相手と対話しながら進められるか |
| 地頭・柔軟性 | 想定外の質問・反論に落ち着いて対応できるか |

> **重要**: 「正解」を当てるゲームではなく、**面接官との共同作業（ディスカッション）**。一人で黙って考え込まず、考えを声に出し、合意を取りながら進める。

## 2. 出題される内容（ケースの種類）

### 2-1. フェルミ推定

実際には調べないとわからない数値を、**論理的な仮定を積み上げて概算**する問題。

- 例：「日本にある電柱の数は？」「東京都内のコンビニの1日の売上は？」「日本のペットボトルの年間消費本数は？」
- ケース本体の前段や、単独の設問として出される。

### 2-2. ビジネスケース（本命）

企業が抱える課題を解決する問題。大きく以下のタイプに分かれる。

| タイプ | 問われること | 例 |
| --- | --- | --- |
| 売上・利益向上 | どうやって伸ばすか | 「あるカフェチェーンの売上を2倍にするには？」 |
| 利益改善・コスト削減 | 利益率をどう上げるか | 「赤字の航空会社を黒字化するには？」 |
| 市場参入 | 参入すべきか/どう参入するか | 「A社は東南アジアに進出すべきか？」 |
| 新規事業 | 何をやるべきか | 「メーカーが新規事業を始めるなら？」 |
| M&A・投資判断 | 買収すべきか | 「B社はC社を買収すべきか？」 |
| 価格設定 | いくらにすべきか | 「新製品の価格をどう決める？」 |

### 2-3. 抽象・公共政策系ケース

ビジネス以外のお題。ロジックの柔軟性を見る。

- 例：「待機児童問題をどう解決するか」「地方の過疎化を止めるには」「日本の食料自給率を上げるには」

## 3. ファーム別の面接形式と傾向

同じ「ケース面接」でも、ファームによって**進め方（誰が主導するか）**と**選考ステップ**が異なります。志望先に合わせて練習形式を変えるのが効率的です。

### 3-1. 外資戦略ファームの2つの型

| 型 | 代表ファーム | 特徴 | 求められる動き |
| --- | --- | --- | --- |
| **面接官リード型**<br>(Interviewer-led) | マッキンゼー | 面接官が設問を細かく区切って順に出す（「まず構造化して」「この数字から何が言える？」等） | 一問一答で**設問ごとに深く正確に**答える。誘導に素直に乗る |
| **候補者リード型**<br>(Candidate-led) | BCG・ベイン ほか多数 | お題だけ渡され、**自分で論点を立てて進める** | 全体構造→優先論点→必要な情報の要求、と**自分で舵を取る** |

- 面接官リード型は「各設問の精度」、候補者リード型は「全体の設計力と推進力」が問われる。
- 迷ったら候補者リード型で練習しておくと両方に対応しやすい（自走の練習は面接官リード型にも活きる）。

### 3-2. 筆記・Webテスト

戦略ファームでは面接前に独自テストが課されることが多い。

- **マッキンゼー**：Solve（旧PST。ゲーム形式の思考力テスト）
- **BCG**：Online Case などのオンライン適性テスト
- **ベイン**：SOVA・玉手箱等のWebテスト（時期・国により変動）

内容・形式は変更されやすいため、**選考体験記で最新の形式を必ず確認**すること。

### 3-3. 日系コンサル・総合商社・事業会社

| 企業タイプ | 出題傾向 |
| --- | --- |
| 日系コンサル（野村総研・アビーム等） | フェルミ推定単体、または30分程度の簡易ケースが中心 |
| 総合商社 | フェルミ推定＋簡単な打ち手立案。地頭とコミュニケーション重視 |
| メガベンチャー・事業会社 | 自社事業に絡めたお題（「うちのアプリのDAUを伸ばすには？」等） |

外資戦略ほどの深さは求められないことが多いが、**構造化と結論ファーストの基本は共通**。志望企業の事業を事前に調べ、事業系のお題に備えておく。

## 4. 解き方の基本フロー（型）

ケースは以下の流れで進めると崩れにくい。

```
① 前提確認・お題の理解
   ↓
② ゴール（何を達成すれば成功か）の定義
   ↓
③ 論点の構造化（フレームワーク／イシューツリー）
   ↓
④ 分析（定量・定性）で仮説を検証
   ↓
⑤ 打ち手の立案・優先順位づけ
   ↓
⑥ 結論（サマリー）
```

### ① 前提確認

- お題の言葉の定義を確認する（「売上とは？」「対象地域は？」「期間は？」）。
- 不明点は面接官に質問してよい（むしろ推奨）。ただし質問攻めにせず、**自分で仮置きして「〜と仮定して進めます」**と宣言するのがスマート。

### ② ゴールの定義

- 「売上を上げる」ではなく「**3年で売上を1.5倍にする**」のように定量的なゴールに落とす。
- ゴールが曖昧なまま進めると打ち手が発散する。

### ③ 構造化

- **イシューツリー**で論点をMECEに分解する。
- フレームワークは"当てはめる"のではなく、**その場のお題に合わせて分解を作る**のが理想。フレームは引き出しとして持っておく（→ 第5章）。

### ④ 分析

- 定量：数式に分解して、どの変数が効くかを特定（感度分析の発想）。
- 定性：顧客・競合・自社の観点で事実を洗う。
- 「どこに問題があるか（ボトルネックの特定）」→「なぜか（原因）」の順で深掘り。

### ⑤ 打ち手

- 複数出し、**インパクト × 実現可能性**で優先順位をつける。
- 「やること」だけでなく「なぜ効くか」「どれくらい効くか」をセットで。

### ⑥ 結論

- **結論ファースト**（PREP：Point→Reason→Example→Point）。
- 「私の結論は〇〇です。理由は3つあり…」と30秒〜1分で言い切る。

## 5. 主要フレームワーク（引き出し）

フレームは万能ではないが、**構造化の出発点**として有効。

| フレームワーク | 用途 | 中身 |
| --- | --- | --- |
| **売上分解** | 売上向上・改善 | 売上 = 客数 × 客単価／客数 = 新規 + 既存 など |
| **利益分解** | 利益改善 | 利益 = 売上 − コスト（固定費・変動費に分ける） |
| **3C** | 市場・戦略全般 | Customer（市場・顧客）/ Competitor（競合）/ Company（自社） |
| **4P** | マーケ施策 | Product / Price / Place / Promotion |
| **バリューチェーン** | コスト・業務分析 | 調達→製造→物流→販売→アフター |
| **PEST** | マクロ環境 | Politics / Economy / Society / Technology |
| **5Forces** | 業界構造 | 競合・新規参入・代替品・売り手・買い手 |
| **AIDMA/AISAS** | 購買行動 | 認知→興味→欲求→記憶→行動 |

### 売上・利益分解の例

```
利益 = 売上 − コスト
売上 = 客数 × 客単価
客数 = 新規顧客 + リピート顧客
客単価 = 1商品単価 × 購入点数
コスト = 固定費（家賃・人件費）+ 変動費（原価・物流）
```

この分解の**どのノードがボトルネックか**を特定してから打ち手を考えるのが定石。

## 6. フェルミ推定の手順

### 基本ステップ

1. **ゴール（求める数）の式を立てる** … 例：市場規模 = 対象人口 × 利用率 × 単価 × 頻度
2. **変数を分解する** … 人口を年齢層・世帯で分けるなど
3. **各変数に数値を仮置き**（→ 覚えておくべき数値が武器になる）
4. **掛け合わせて計算**（桁を間違えないよう概数で）
5. **妥当性チェック**（オーダーが常識的か。ズレていたら仮定を見直す）

### 例：日本のコンビニ市場規模（年間）

```
店舗数 約56,000店
1店舗あたり1日の来客数 約1,000人
客単価 約700円
→ 1日売上 = 56,000 × 1,000 × 700 = 約392億円/日
→ 年間 = 392億 × 365 ≒ 約14兆円
```

（実際のコンビニ業界売上は約11〜12兆円規模。オーダーが合っていればOK。）

### コツ

- **アプローチ**（需要側から/供給側から）を宣言してから始める。
- 人口・世帯数など**基礎数値を暗記**しておくと速い（→ [数値まとめ](case-interview-key-numbers.md)）。
- 計算は概数（四捨五入・切りのいい数字）で。桁数だけは絶対に間違えない。

## 7. 例題と模範解答

型を頭に入れたら、実際の解答の流れを見るのが一番早い。数値は [数値まとめ](case-interview-key-numbers.md) の暗記値を使っています。

### 7-1. ビジネスケース例題：「あるカフェチェーンの売上を2倍にするには？」

第4章のフローに沿ったフル解答例。

**① 前提確認（〜1分）**

> 「前提を確認させてください。対象は国内展開の中規模チェーン（100店舗程度）、期間は3年、売上は既存事業ベースで考えてよいでしょうか？ 新規事業やM&Aは一旦スコープ外とします。」

**② ゴール定義**

> 「現在の1店舗あたり年商を約5,000万円と仮定すると、チェーン全体で年商約50億円。これを3年で100億円にすることをゴールとします。」

**③ 構造化**

```
売上 = 店舗数 × 1店舗あたり売上
1店舗あたり売上 = 客数 × 客単価
客数 = 席数 × 回転率 × 営業時間帯構成（＋テイクアウト）
客単価 = ドリンク単価 + フード・物販の付加
```

> 「伸ばすレバーは大きく『店舗数を増やす』『1店舗の客数を増やす』『客単価を上げる』の3つです。まずどこに伸びしろがあるか確認させてください。」

**④ 分析（ボトルネック特定）**

面接官から「昼過ぎ〜夕方の稼働が低い」「客単価は業界平均並みの600円」という情報を得たとする。

> 「ピーク（朝・昼）は席が埋まっているとすると、客数の伸びしろは**アイドルタイム（14〜18時）**にある。また客単価はフード付帯率を上げる余地がある、と整理できます。」

**⑤ 打ち手（インパクト × 実現性で優先順位）**

| 打ち手 | 狙うレバー | インパクト | 実現性 |
| --- | --- | --- | --- |
| アイドルタイム向け商品・割引（カフェタイムセット） | 客数 | 中 | 高 |
| モバイルオーダー・テイクアウト強化 | 客数（席数の制約を外す） | 大 | 中 |
| フードのセット提案・レジ横物販 | 客単価 | 中 | 高 |
| 出店加速（居抜き・小型店） | 店舗数 | 大 | 中 |

**⑥ 結論（結論ファースト）**

> 「結論として、短期はアイドルタイム集客とフード付帯で1店舗売上を約1.3倍にし、並行して小型店フォーマットで3年で店舗数を1.5倍にする、この掛け算で売上2倍を狙います。理由は、既存店改善は投資が小さく即効性がある一方、2倍という水準は店舗数の拡大なしには届かないためです。」

### 7-2. ケースタイプ別の解答骨子

第2章の出題タイプごとに、構造化と結論までの「骨子」を示す。本番ではこれを対話しながら肉付けしていく。

#### タイプ①：市場参入 ——「飲料メーカーA社は東南アジアに進出すべきか？」

```
判断基準 = 市場の魅力度 × 自社の勝ち筋 × 参入方式の成立性
├ 市場の魅力度：市場規模と成長率（人口6.8億人・経済成長・中間層拡大）、競合状況
├ 自社の勝ち筋：既存ブランド・製品が現地ニーズに合うか、価格帯、販路
└ 参入方式：自前進出／現地パートナーとのJV／買収 —— スピードとリスクの比較
```

**結論例**：「進出すべきと考えます。市場は人口・成長率とも魅力的で、自社の主力製品は現地の◯◯需要に合致します。ただし販路構築が最大のボトルネックのため、初期は現地流通大手とのJVでリスクを抑えた参入を提案します。」

- ポイント：**「べきか」型は判断基準を先に立てる**。Yes/Noを最後まで曖昧にしない。

#### タイプ②：利益改善 ——「赤字の航空会社を黒字化するには？」

```
利益 = 売上 − コスト
├ 売上 = 路線数 × 便数 × 座席数 × 搭乗率 × 単価 → 路線別の採算を分解
└ コスト = 変動費（燃料・空港使用料） + 固定費（機材リース・人件費・整備）
```

主要論点：①どの路線が赤字か（不採算路線の特定）→ ②搭乗率と単価に打ち手はあるか（ダイナミックプライシング・貨物混載）→ ③固定費構造を変えられるか（機材統一・リース見直し）。

**結論例**：「不採算のローカル3路線の減便・撤退で赤字の6割を止め、残りは機材の小型化で搭乗率を70%→85%に引き上げて黒字化します。」

- ポイント：**「どこで赤字が出ているか」の特定が先**。打ち手はその後。

#### タイプ③：M&A ——「B社はC社を買収すべきか？」

```
判断基準：
① 戦略適合：自社戦略の何を埋める買収か（技術・顧客・地域・人材）
② 単独価値：C社は単体で儲かっているか（売上・利益・成長性）
③ シナジー：買収で追加される価値（クロスセル・コスト統合）を定量化
④ 価格とリスク：価格 < 単独価値+シナジー か。統合失敗・人材流出リスク
```

**結論例**：「買収すべきです。C社の顧客基盤は自社の弱い中小企業セグメントを補完し、販路シナジーだけで年◯億円の増益が見込めます。ただし想定価格が単独価値＋シナジーを上回るなら見送るべきで、上限は◯◯億円です。」

- ポイント：**「いくらまでなら買ってよいか」まで踏み込む**と評価が高い。

#### タイプ④：公共政策 ——「待機児童問題をどう解決するか？」

```
待機児童 = 保育需要 − 保育供給
├ 需要側：共働き世帯の増加、都市部への集中 → 需要を分散・平準化できるか
└ 供給側：施設数 × 定員。ボトルネックは「場所」より「保育士不足」
```

主要論点：①どこで（都市部）・なぜ（供給不足）ギャップが生じているか → ②供給の制約は施設か人か → ③施策を「即効性（小規模保育・空きスペース活用）」と「構造改革（保育士処遇改善・配置基準）」に分ける。

**結論例**：「最大のボトルネックは都市部の保育士不足です。短期は小規模保育と保育士の処遇改善で供給を増やし、中期は企業内保育やリモートワーク普及で需要側の集中も緩和する、供給・需要の両面策を提案します。」

- ポイント：ビジネスケースと同じく**需給・因数分解で構造化**できる。「誰にとっての問題か（子ども・親・行政）」の視点整理も有効。

### 7-3. フェルミ推定の例題

#### 例題A：日本にある電柱の数は？

```
面積アプローチ：
日本の面積 約38万km²、うち人が住む可住地は約3割 ≒ 11万km²
電柱はおよそ50m間隔で立っている
→ 1km²あたり (1,000m ÷ 50m)² = 400本
→ 11万km² × 400本 = 4,400万本
過疎地は間隔が広いことを踏まえ、約3,000万〜4,000万本と推定
```

（実際は約3,600万本。オーダーが合っていればOK。）

#### 例題B：スターバックス1店舗の1日売上は？

```
座席 約50席、営業 約15時間
ピーク4時間は満席で1時間1回転 → 50 × 4 = 200人
オフピーク11時間は稼働3割・1時間1回転 → 15 × 11 ≒ 165人
テイクアウトを店内の5割と置く → (200+165) × 1.5 ≒ 550人
客単価 約700円（ドリンク＋フード）
→ 550人 × 700円 ≒ 約40万円/日（年間 約1.4億円）
```

#### 例題C：日本のペットボトル飲料の年間消費本数は？

```
人口 約1.2億人
1人あたり 平均0.5本/日（毎日1本の人と、ほぼ飲まない人の平均）と仮定
→ 1.2億 × 0.5 × 365 ≒ 約220億本/年
```

（実際の清涼飲料PETボトル出荷は年200億本超の規模。）

### 7-4. 良い回答・NG回答の対比

お題：「駅前の書店の売上を上げるには？」

**NG回答（いきなり打ち手・羅列型）**

> 「えっと、SNSで宣伝するとか、カフェを併設するとか、イベントをやるとか…あとポイントカードとか…」

- 構造がなく、なぜその打ち手なのか根拠がない。網羅性も検証できない。

**良い回答（構造 → 特定 → 打ち手）**

> 「まず売上を客数×客単価に分けます。駅前という立地から通行量＝潜在客数は多いはずなので、論点は『入店率』か『購入率・単価』のどちらが弱いかだと考えます。仮に入店率が低いなら、店頭の見せ方や品揃えの訴求を…（面接官と対話しながら特定を進め）…結論として、最も効くのは◯◯です。理由は…」

- 分解を宣言 → 弱い箇所を対話で特定 → 根拠つきで打ち手、の順になっている。

---

## 8. よくある失敗と対策

| 失敗 | 対策 |
| --- | --- |
| いきなり打ち手を並べる | 必ず構造化→原因特定→打ち手の順で |
| フレームに当てはめて満足 | フレームは分解の出発点。お題に合わせて調整する |
| 黙って考え込む | 考えを声に出す。詰まったら「少し整理させてください」 |
| MECEでない分解 | 漏れ・ダブりを自分でチェック。「他には？」と自問 |
| 数字の桁ミス | 単位を明示（万・億・兆）。概算後にオーダー確認 |
| 結論が曖昧 | 最後に必ず結論ファーストでサマリー |
| 面接官の誘導を無視 | ヒントは仮説修正のサイン。素直に取り込む |

## 9. 対策・練習方法

### 9-1. インプット（基礎固め）

- **定番書籍**で型を学ぶ。
  - 『東大生が書いた 問題を解く力を鍛えるケース問題ノート』
  - 『地頭力を鍛える』（フェルミ推定の古典）
  - 『考える技術・書く技術』（ロジカルシンキング／ピラミッド原則）
  - 『ロジカル・シンキング』（照屋・岡田）
- 各社の選考体験記（ES・面接情報）で**過去の頻出お題**を把握。

### 9-2. アウトプット（実戦練習）

- **1人練習**：お題を1つ決め、紙に構造化→5分で結論まで書く。時間を計る。お題は[頻出お題集（50問）](case-interview-question-bank.md)から選ぶとよい。
- **模擬面接（ペア練習）**が最重要。**面接官役 ⇄ 回答者役**を交代し、フィードバックし合う。会話の進め方は[模擬面接フル対話スクリプト](case-interview-mock-script.md)が参考になる。
  - 就活コミュニティ、キャリア支援サービス、OB/OG、内定者などを活用。
- 練習後は**録画・録音**して、「結論から話せているか」「沈黙が長くないか」を振り返る。

### 9-3. 日常トレーニング

- 街を歩きながら「この店の1日の売上は？」と**フェルミ推定を習慣化**。
- ニュースの企業戦略を「自分ならどう分析するか」で考える。
- **数字の暗記**（人口・GDP・主要市場規模）を進める（→ [数値まとめ](case-interview-key-numbers.md)）。

### 9-4. 直前・当日

- 結論ファースト（PREP）を体に染み込ませる。
- 面接官との**対話**を意識（独演会にしない）。
- わからなくても堂々と。仮定を置いて前に進める姿勢が評価される。

## 10. 対策スケジュール例（3ヶ月ロードマップ）

面接本番から逆算した3ヶ月プランの例。社会人・学業と並行する前提で、1日30分〜1時間＋週末の模擬面接を想定。

| 時期 | フェーズ | やること | 目安量 |
| --- | --- | --- | --- |
| 1ヶ月目 前半 | インプット | 定番書籍で型を学ぶ（ケース問題ノート・地頭力を鍛える） | 2冊読了 |
| 1ヶ月目 後半 | インプット | フレームワーク整理＋[基礎数値の暗記](case-interview-key-numbers.md)開始 | 数値カテゴリ1〜2個/日 |
| 2ヶ月目 前半 | 1人練習 | フェルミ推定を毎日1問（紙に書いて5分で結論まで） | 毎日1問 |
| 2ヶ月目 後半 | 1人練習 | ビジネスケースを構造化→打ち手→結論まで通しで解く | 週2〜3問 |
| 3ヶ月目 | 模擬面接 | ペア練習（面接官役⇄回答者役を交代）。録音して振り返り | 週2回以上 |
| 直前1週間 | 総仕上げ | 志望ファームの形式（第3章）に合わせた通し練習、頻出お題の復習 | 毎日1セット |
| 前日 | 最終確認 | 結論ファースト（PREP）の口慣らし、基礎数値の再確認。夜更かししない | 軽めに |

### 進め方のポイント

- **模擬面接の開始を遅らせない**こと。1人練習だけでは「対話しながら進める」力がつかないため、3ヶ月目を待たず2ヶ月目から週1回混ぜてもよい。
- 各練習後に「構造化できたか／結論から話せたか／数字の桁を間違えなかったか」の3点だけ振り返る（第8章の失敗リストをチェックリスト代わりに）。
- 期間が1ヶ月しかない場合は、インプットを1週間に圧縮し、残りを1人練習＋模擬面接に充てる。
