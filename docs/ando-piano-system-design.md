# 低トラフィックサイト向けシステム構成ガイド
## ando-piano.jp リニューアル設計

対象規模: アクセス数が少ない日100人未満（月2,000〜3,000 PV程度）  
用途: 習い事教室サイト（複数サイト運営前提）

---

## 推奨システム構成

### アーキテクチャ全体図

```
【コンテンツ更新】
  Decap CMS（ブラウザ）
       ↓ Git commit
  GitHub（各サイトのリポジトリ）
       ↓ Webhook
【ビルド・ホスティング】
  Cloudflare Pages（全サイト）
       ↓
【フォーム送信】
  Cloudflare Turnstile（スパム対策）
       ↓
  Cloudflare Workers（サーバーレス処理）
       ↓
  Brevo API（メール送信）
       ↓
  教室オーナーのメール受信
```

## 各コンポーネント詳細

### Cloudflare Pages（ホスティング）

- プロジェクト数: **無制限**
- リクエスト数: **無制限**
- 帯域幅: **無制限**
- ビルド回数: 500回/月（無料）
- 商用利用: **OK**

Vercel Hobbyプランは商用利用禁止のため、習い事サイトはCloudflare Pagesを選択。

### Decap CMS + GitHub（コンテンツ管理）

- ブラウザ上で記事・画像を編集（WordPressと同等の操作感）
- 保存するとGitHubにコミット → 自動ビルド → 約1〜2分で反映
- サイト数・コンテンツ数・ユーザー数すべて無制限・無料
- 各サイトのGitHubリポジトリに設定ファイルを置くだけ

MicroCMSは無料プランだと1サービスのみのため、複数サイト運営にはDecap CMSが最適。

### Cloudflare Workers（フォーム処理）

- リクエスト数: 10万回/日（無料）
- フォーム送信 → Brevo API経由でメール送信

### Brevo（メール送信）

- 送信数: **300通/日（全サイト合計）**
- 送信ドメイン数: 複数OK
- 月額: **¥0**

## 保守・運用面

### 非エンジニアによる日常更新

```
Decap CMS管理画面
  ├── お知らせ（タイトル・本文・日付・カテゴリ）
  ├── ギャラリー（画像アップロード）
  └── スケジュール（発表会・レッスン情報）
```

更新後は自動ビルド・自動デプロイ（約1〜2分で反映）。

### セキュリティ

- 静的サイトのためDBハック・SQLインジェクションのリスクがゼロ
- WordPressのような脆弱性スキャン・プラグイン管理が不要
- 依存パッケージはGitHub Dependabotで自動アラート

### バックアップ

- コードはGitHubで管理（履歴が自動バックアップ）
- 画像・コンテンツもGitHub上に保存
