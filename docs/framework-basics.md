# WebフレームワークとはなにかまたExpress・Railsとは

---

## フレームワークとは何か

**よく使う機能をあらかじめまとめた「開発の土台」のこと。**

家を建てるとき、柱・基礎・屋根をゼロから作るより、ある程度出来上がった「建売の骨組み」に内装だけ自分でつけた方が速い。フレームワークはその骨組みにあたる。

```
フレームワークなし → 全部自分で書く（大変・時間がかかる）
フレームワークあり → 共通部分は用意されている（本来の機能だけ書けばいい）
```

---

## Webの仕組みとサーバーの役割

```
ユーザー（ブラウザ）
    ↓ 「このページを見せて」とリクエスト
サーバー（Express / Rails 等）
    ↓ データベースを読み書きして処理する
データベース（MySQL・PostgreSQL 等）
    ↓ 結果をHTMLやJSONで返す
ユーザー（ブラウザに表示される）
```

ExpressとRailsはどちらも**この「サーバー」の役割を担うフレームワーク**。

---

## Express とは

**Node.js（JavaScript）で動く軽量Webサーバーフレームワーク。**

Node.jsはブラウザ以外（サーバー側）でJavaScriptを動かす仕組み。
Expressはそれをさらに使いやすくしたもの。

### 最小限のコード例

```javascript
const express = require('express')
const app = express()

// "/" にアクセスしたら「こんにちは！」を返す
app.get('/', (req, res) => {
  res.send('こんにちは！')
})

// 3000番ポートで待ち受け開始
app.listen(3000)
```

これだけで `http://localhost:3000` にアクセスできるサーバーができる。

### 特徴

| 項目 | 内容 |
|---|---|
| 言語 | JavaScript（Node.js） |
| 規模感 | 軽量・最小限 |
| 自由度 | 高い（構成を自分で決める） |
| 向いている用途 | REST API・小〜中規模バックエンド |
| 学習コスト | 低め（JSを知っていれば始めやすい） |

### よく使われる場面

- フロントエンド（React・Vue等）と組み合わせるAPIサーバー
- スマホアプリのバックエンド
- 小規模なWebサービスのバックエンド

---

## Rails（Ruby on Rails）とは

**Ruby言語で動くフルスタックWebフレームワーク。**

「フルスタック」＝ 画面（HTML）・サーバー処理・データベース管理まで**全部まとめて面倒を見てくれる**。

### コマンド一発で雛形が生成される

```bash
rails new myapp        # アプリの骨組み一式が自動生成
rails generate scaffold Post title:string body:text
                       # 記事の一覧・詳細・作成・編集・削除画面が自動生成
rails db:migrate       # データベースのテーブルを作成
rails server           # サーバー起動
```

`scaffold` コマンド1つで、一覧・詳細・新規作成・編集・削除の画面とAPIが全部できあがる。

### 「設定より規約」の思想

Railsは決まりごと（規約）に従って開発する。迷う場面が少なく速く作れる。

```
例：モデル名を「Post」にすると、
    テーブル名は自動で「posts」になる
    ファイルの置き場所も自動で決まる
    → 自分で設定しなくていい
```

### 特徴

| 項目 | 内容 |
|---|---|
| 言語 | Ruby |
| 規模感 | フル装備（画面〜DB管理まで一体） |
| 自由度 | 低め（Railsの流儀に従う） |
| 向いている用途 | Webサービス全般・管理画面・ブログ・予約サイト等 |
| 学習コスト | 中程度（Rubyの学習が必要） |

### よく使われる場面

- スタートアップの初期プロダクト（速く作れるため）
- 管理画面付きのWebサービス
- ブログ・ECサイト・予約サイト等

---

## Express と Rails の比較

| | Express | Rails |
|---|---|---|
| 言語 | JavaScript（Node.js） | Ruby |
| タイプ | 軽量・最小限 | フルスタック |
| 自動生成 | ほぼなし（自分で設計） | コマンドで大量生成 |
| 自由度 | 高い | 低め（規約に従う） |
| 速く作れるか | △（設計が必要） | ○（規約で迷わない） |
| 学習コスト | 低め | 中程度 |
| 向いている人 | JS経験者・API開発 | Webサービスを速く作りたい人 |

---

## 他の主なフレームワークとの位置づけ

| フレームワーク | 言語 | タイプ | Expressとの比較 |
|---|---|---|---|
| **Express** | JavaScript | 軽量 | 基準 |
| **NestJS** | TypeScript | フル | Expressの上に構造を加えたもの |
| **FastAPI** | Python | 軽量 | Python版Express的な位置づけ |
| **Django** | Python | フル | Python版Rails的な位置づけ |
| **Spring Boot** | Java/Kotlin | フル | Java版・大規模向け |
| **Laravel** | PHP | フル | PHP版Rails的な位置づけ |

---

## どちらを選ぶべきか

```
JavaScriptで書きたい・APIだけ作りたい
    → Express（またはNestJS・Hono等）

Rubyで書いてもいい・Webサービスを速く作りたい
    → Rails

Pythonを使いたい
    → FastAPI（API中心）or Django（フルスタック）

Javaで大規模システム
    → Spring Boot
```
