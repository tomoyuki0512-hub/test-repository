# 行動面接（フィット面接）対策ガイド

コンサル・外資系の選考は**ケース面接＋フィット面接（行動面接 / Behavioral）のセット**が基本です。ケースが完璧でもフィットで落ちることは普通にあります。ここでは頻出質問と答え方の型（STAR法）、準備の仕方をまとめます。

> ケース面接側の対策は [ケース面接 完全ガイド](case-interview-guide.md) を参照。

---

## 1. フィット面接の位置づけ

| 観点 | 内容 |
| --- | --- |
| 何を見ているか | 一緒に働きたいか、クライアントの前に出せるか、成長ポテンシャル、価値観の一致 |
| 重さ | ケースと**同等以上**。特にマッキンゼーの **PEI**（Personal Experience Interview）は、1つのエピソードを15〜20分深掘りする独立パート |
| ケースとの違い | 「正しい答え」ではなく「**あなたが実際にどう考え、動いたか**」の事実を聞かれる |

> **PEIの特徴**：「リーダーシップを発揮した経験を1つ」と聞かれた後、「そのとき相手は何と言った？」「なぜその選択をした？」と**行動レベルまで**掘られる。話を盛ると必ず破綻するので、実体験を精緻に思い出しておくことが最大の対策。

---

## 2. STAR法（回答の型）

エピソードは以下の順で話すと構造的に伝わる。

| 要素 | 内容 | 時間配分の目安 |
| --- | --- | --- |
| **S**ituation | 状況（いつ・どこで・誰と・何が起きていたか） | 15% |
| **T**ask | 課題（何が求められ、何が難しかったか） | 15% |
| **A**ction | **自分が**取った行動（ここが本体。「私たちは」でなく「私は」） | **50%** |
| **R**esult | 結果（できれば数字で）＋そこから学んだこと | 20% |

### 良い回答の条件

- **Actionが主語「私」で具体的**（役職や立場ではなく、実際の言動）
- 結果に**数字**がある（「売上◯%増」「参加者が2倍」「離脱ゼロに」）
- 困難・対立・失敗が含まれる（順風満帆な話は評価されない）
- 1回の回答は**2〜3分**。深掘りは面接官に委ねる

## 3. 頻出質問20と答え方の骨子

### A. 志望動機系

| # | 質問 | 答え方の骨子 | NG例 |
| --- | --- | --- | --- |
| 1 | なぜコンサルタントになりたいのか | 原体験→コンサルで得られるもの（課題解決の型・産業横断の経験）→将来像、を一本の線で | 「成長したいから」だけ（どの仕事でも言える） |
| 2 | なぜ当社か（他ファームとの違い） | そのファームの案件領域・カルチャー・出会った社員の具体名/言葉を根拠に | HPの受け売り（「グローバルだから」） |
| 3 | 他にどこを受けているか | 正直に。軸が一貫していることを示す | 場当たり的な業界の羅列 |
| 4 | コンサルの仕事の厳しい面をどう考えるか | 理解した上で、それでもやる理由（過去の類似経験）を添える | 「大丈夫です」の根拠なし |
| 5 | 5年後・10年後どうなっていたいか | 手段（役職）でなく「何を解ける人になりたいか」で語る | 「パートナーになりたい」だけ |

### B. リーダーシップ・チーム系（PEI頻出）

| # | 質問 | 答え方の骨子 | NG例 |
| --- | --- | --- | --- |
| 6 | リーダーシップを発揮した経験 | 役職でなく**影響力**の話。反対者・非協力者をどう動かしたか | 「部長を務めました」（役職の説明で終わる） |
| 7 | チーム内の対立をどう解決したか | 双方の利害を構造化→共通のゴールに再設定した過程 | 「話し合いで解決」（何をしたか不明） |
| 8 | 価値観の違う人と働いた経験 | 相手の前提を理解しに行った行動を具体的に | 相手が折れた話（自分が動いていない） |
| 9 | 人を説得した経験 | データ/相手の関心に合わせた伝え方の工夫 | 熱意で押し切った話 |
| 10 | 周囲を巻き込んで成し遂げたこと | 「誰に・何を・どう依頼したか」の設計を語る | 「みんなで頑張った」 |

### C. 困難・挫折系

| # | 質問 | 答え方の骨子 | NG例 |
| --- | --- | --- | --- |
| 11 | 一番の挫折経験 | 失敗の事実→原因の自己分析→行動を変えた→その後の結果 | 挫折と呼べない軽い話／他責 |
| 12 | 目標を達成できなかった経験 | 何が読み違いだったかを構造的に振り返る | 「環境が悪かった」 |
| 13 | 批判・厳しいフィードバックを受けた経験 | 受け止め方と、実際に変えた行動 | 「気にしませんでした」 |
| 14 | プレッシャーの中で成果を出した経験 | 優先順位づけ・タスク分解で乗り切った過程 | 根性論のみ |

### D. 自己理解系

| # | 質問 | 答え方の骨子 | NG例 |
| --- | --- | --- | --- |
| 15 | 強みは何か | 強み→それが発揮された具体例→コンサルでどう活きるか | 抽象語だけ（「論理的思考力です」） |
| 16 | 弱みは何か | 本当の弱み＋改善のために実際にしていること | 強みの言い換え（「頑張りすぎる」） |
| 17 | 周囲からどんな人と言われるか | 他者評価と自己認識のギャップも語れると深い | 「優しい人」で終わる |
| 18 | 学生時代（前職）で最も打ち込んだこと | STARで。数字と自分の役割を明確に | 事実の羅列（成果・学びがない) |
| 19 | 最近気になったニュース | 事実→自分の視点（構造化・示唆）→立場を言い切る | 要約だけで意見がない |
| 20 | 何か質問はありますか（逆質問） | 下の逆質問集を参照。「特にありません」は避ける | 調べればわかる質問（福利厚生等） |

## 4. エピソードの棚卸しワークシート

面接前に、**代表エピソードを3つ**STARで書き出しておく。1つのエピソードは複数の質問（リーダーシップ/困難/巻き込み）に使い回せる。

| | エピソード1 | エピソード2 | エピソード3 |
| --- | --- | --- | --- |
| テーマ（例） | リーダーシップ | 挫折→立て直し | 対立の解消 |
| Situation | | | |
| Task（何が難しかったか） | | | |
| Action（私が何をしたか） | | | |
| Result（数字＋学び） | | | |
| 想定深掘り質問への答え | | | |

**深掘り想定**：「なぜそうした？」「他の選択肢は？」「相手はなんと言った？」「今ならどうする？」——各エピソードにつきこの4つは必ず答えられるようにしておく。

---

## 5. 逆質問集

### 良い逆質問の例

- 「入社1年目のコンサルタントが最初につまずきやすいのはどんな点ですか？」
- 「◯◯さんがこのファームに残り続けている理由は何ですか？」
- 「最近のプロジェクトで、チームの議論が最も白熱したのはどんな論点でしたか？」
- 「ケース面接で私の弱かった点があれば、率直に教えていただけますか？」（学習意欲を示せる）

### 避けるべき逆質問

- 調べればわかること（残業時間・給与・福利厚生の一般論）
- Yes/Noで終わる質問
- 「特にありません」

---

## 6. 準備チェックリスト

- [ ] 「なぜコンサル」「なぜ当社」を1分で言える
- [ ] 代表エピソード3つをSTARで書き出した（ワークシート）
- [ ] 各エピソードで深掘り4質問（なぜ/他の選択肢/相手の反応/今なら）に答えられる
- [ ] 強み・弱みを具体例つきで言える
- [ ] 結果を数字で言える箇所を確認した
- [ ] 逆質問を2つ以上用意した
- [ ] 声に出して2〜3分に収まるか計測した（ケース練習の録音と同じ要領）
