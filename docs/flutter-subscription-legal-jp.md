# サブスクリプション 法務・表記ガイド（日本向け実務チェックリスト）

日本でサブスクリプションアプリを提供する際に確認すべき法令・表記の**実務チェックリスト**です。

> ⚠️ **免責:** 本ガイドは法的助言ではなく、開発・企画担当者が論点を把握するための整理です。実際の表記・規約は**必ず弁護士等の専門家のレビュー**を受けてください。法改正も頻繁にあります（更新日: 2026-07-13）。

---

## 1. 前提：誰が「販売者」か（ストアによる違い）

法的義務の整理でまず重要なのが、**IAPの売買契約の当事者がストアによって異なる**点です。

| | iOS（App Store） | Android（Google Play） |
| --- | --- | --- |
| 販売者（Merchant of Record） | **Apple**（開発者はAppleに販売、Appleがユーザーに販売する構造） | **開発者**（Googleは決済代行的な立場） |
| 返金の主体 | Apple | 開発者（＋Google） |
| 領収書の発行 | Apple | Google Play（開発者名義） |

- Google Play では開発者が販売者となるため、**特定商取引法の「通信販売」表記義務は開発者側にある**と整理するのが安全です。
- App Store でも、アプリ内の表示・広告表現には景品表示法等が及ぶため、「Appleが売主だから表記不要」と単純化しないこと。

## 2. 特定商取引法（特商法）

### 2-1. 「特定商取引法に基づく表記」ページ

通信販売にあたるため、以下を記載したページを用意し、**アプリ内とWebサイトから到達できる**ようにします。

- [ ] 事業者名・代表者名・所在地・連絡先
- [ ] 販売価格（税込）・価格以外の必要料金
- [ ] 支払方法・支払時期（例:「初回購入時。以降は各更新日にストア経由で自動課金」）
- [ ] 役務の提供時期（例:「決済完了後ただちに利用可能」）
- [ ] **解約方法・解約の効果**（ストアのサブスク管理画面での操作手順、期間途中の解約は次回更新の停止であること）
- [ ] 動作環境（対応OSバージョン）

### 2-2. 2022年改正：定期購入の「最終確認画面」表示義務

いわゆる**詐欺的定期購入商法対策**の改正（2022年6月施行）で、申込みの最終確認画面に以下を明示することが義務化されました（違反は行政処分・取消権の対象）。

| 表示義務項目 | サブスクアプリでの実務 |
| --- | --- |
| 分量・提供期間 | 「月額プラン（1ヶ月ごと自動更新）」を明記 |
| 価格・対価 | 税込価格。**無料トライアル後の本課金額**を必ず併記 |
| 支払時期・方法 | 「トライアル終了日の24時間前までに解約しない場合、◯月◯日に¥980が請求されます」等 |
| 解除（解約）の方法 | 解約手順・リンクを申込み画面に表示 |
| 申込期間（期限があれば） | キャンペーン終了日等 |

**実務上の注意:** ストアの決済シート（Apple/Googleが表示）だけに頼らず、**その手前のアプリ内ペイウォール画面**に上記を表示するのが安全です。これは[ストア審査要件（Apple Guideline 3.1.2等）](flutter-subscription-guide.md)とも重なります。→ 画面例は [UX設計ガイド 第6章（課金タイミングの表記）](flutter-subscription-ux.md)

## 3. 景品表示法（表示・広告）

- [ ] 「**無料**」訴求には**自動更新で有料に移行する旨の打消し表示**を近接・同等の視認性で（有利誤認の防止）
- [ ] 「業界No.1」「最安」等の優良誤認になり得る表現は根拠を確保
- [ ] 価格の二重表示（「通常¥1,200→¥980」）は比較対照価格の要件を確認
- [ ] ステルスマーケティング規制（2023年10月〜）: 広告であることを隠したレビュー・SNS投稿の依頼はNG

---

## 4. 消費者契約法・ダークパターン回避

- [ ] 解約を不当に妨げる設計（解約導線を隠す・引き留め画面の連打）は**消費者契約法上のリスクかつストアポリシー違反**。→ [UX設計ガイド 第8章（やってはいけないこと）](flutter-subscription-ux.md)
- [ ] 利用規約に「いかなる場合も返金しない」等の**消費者の利益を一方的に害する条項**は無効になり得る（ストアの返金制度と矛盾する記載もNG）
- [ ] 未成年者の利用: 年齢確認・保護者同意の考え方を規約に明記（未成年者取消しへの備え）

## 5. その他の法令・規約

| 論点 | 要点 |
| --- | --- |
| **個人情報保護法（APPI）** | プライバシーポリシーの整備（**ストア審査の必須要件**でもある）。購入履歴・識別子の利用目的を明記。第三者提供・委託（分析SDK等）の整理 |
| **資金決済法** | 自動更新サブスク自体は通常対象外。ただし**アプリ内コイン等の前払式支払手段**を併用する場合は表示義務・供託等の検討が必要 |
| **ストア規約との整合** | 利用規約に「解約・返金はストアの規定に従う」旨を明記し、自社規約とストア規約の矛盾をなくす。EULA（iOS: AppleのStandard EULAを使うか自社EULAか）を確認 |
| **プラットフォーム関連法制** | スマホソフトウェア競争促進法（日本・2025年12月施行）により代替決済等の環境が変化中。外部決済を採用する場合は特商法・資金決済法の整理が**すべて自社責任**になる点に注意 |
